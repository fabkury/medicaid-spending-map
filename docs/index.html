Methodology

Data Source

This map visualizes the Medicaid Provider Spending dataset published by the U.S. Department of Health & Human Services (HHS), Centers for Medicare & Medicaid Services (CMS). The dataset contains 238 million rows covering 21.8 billion claim lines from January 2018 through December 2024, representing approximately $21.8 trillion in total Medicaid payments. Each row aggregates claims by billing provider, servicing provider, HCPCS code, and month.

HCPCS Code Selection

The dataset contains 12,263 unique HCPCS (Healthcare Common Procedure Coding System) codes. To keep the interactive map performant, only the top 35% most popular codes by claim volume are included (3,893 codes). These codes account for 99.8% of all claim lines and over 99.9% of all dollars paid. Code descriptions are sourced from the CMS 2026 Physician Fee Schedule Relative Value Unit (RVU) file.

Mapping Providers to Counties

The spending dataset identifies providers by National Provider Identifier (NPI) numbers but does not include geographic information. To map spending to counties, a three-step lookup is performed:

  1. NPI → ZIP code: Each NPI is linked to its 5-digit practice location ZIP code using the NPPES (National Plan and Provider Enumeration System) registry. This yielded 9.1 million NPI-to-ZIP mappings.
  2. ZIP code → County FIPS: ZIP codes are mapped to county FIPS codes using the Census Bureau ZCTA-to-County Relationship File. When a ZIP code spans multiple counties, the county with the largest land area overlap is selected.
  3. County FIPS → Map polygon: County boundaries are rendered using the U.S. Census Bureau's cartographic boundary shapefiles (2020 vintage, simplified to 1:10m resolution via TopoJSON).

Billing vs. Servicing NPI

Each row in the spending dataset may contain two NPI numbers:

  • Billing NPI — the entity that submitted the claim (present in 96.7% of rows, covering 87.4% of claims).
  • Servicing NPI — the provider who delivered the service (present in 96.2% of rows, covering 83.1% of claims).
  • Both NPIs present — 93.0% of rows (80.0% of claims) have both.

For geographic assignment, the servicing provider’s location is used when available, with the billing provider’s location as a fallback. This reflects where the care was delivered rather than where the claim was filed.

ZIP Code Agreement

Among the 218 million rows where both NPIs could be resolved to a ZIP code via NPPES:

  • 61.3% of rows (76.5% of claims) had identical billing and servicing ZIP codes.
  • 38.7% of rows (23.5% of claims) had different ZIP codes.
  • Even when ZIP codes differed, 74.3% of rows (84.6% of claims) mapped to the same county, since multiple ZIP codes often fall within a single county.

County Resolution Rate

Using the servicing-then-billing NPI fallback strategy:

  • 98.8% of rows (88.9% of claims) were successfully mapped to a county.
  • The remaining 1.2% of rows (11.1% of claims) could not be geolocated because neither NPI was found in the NPPES registry or the associated ZIP code did not appear in the Census ZCTA crosswalk. These claims are excluded from the map.

Per-Capita Calculation

Dollar values on the map are per-capita amounts, computed as:

Per-capita spending = Total paid in county for code & quarter ÷ County population

The TOTAL_PAID field from the spending dataset is summed by county, HCPCS code, and calendar quarter. County population estimates come from the U.S. Census Bureau’s 2024 population estimates (co-est2025-alldata). A single population figure (2024 estimate) is used across all quarters.

Quarterly Aggregation

The source data is monthly. Claims are grouped into calendar quarters (Q1 = Jan–Mar, Q2 = Apr–Jun, Q3 = Jul–Sep, Q4 = Oct–Dec) based on the CLAIM_FROM_MONTH field. The dataset spans Q1 2018 through Q4 2024 (28 quarters).

Color Scale

The choropleth uses a sequential blue color scale with linear interpolation. To prevent extreme outliers from compressing the visible color range, the scale maximum is set at the 98th percentile of per-capita values across all counties for the selected code and quarter. Counties above this threshold are shown in the darkest blue. Counties with no data for the selected code/quarter are shown in gray.

Limitations

  • Provider location is used as a proxy for where services were delivered. In some cases (e.g., telehealth, lab services), the provider’s registered address may differ from the patient’s location.
  • When a ZIP code spans multiple counties, only the county with the largest land-area overlap is used, which may misattribute some spending near county borders.
  • A fixed 2024 population estimate is used for all quarters (2018–2024), which does not account for population changes over time.
  • 11.1% of claims by volume could not be mapped to a county and are excluded.
